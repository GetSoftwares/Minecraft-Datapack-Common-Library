# **术语**

## **基本标签**

具体而言，基本标签包含了以下标签：

`common:tags/damage_type/vanilla_all_damage_types`（伤害类型标签）
`common:tags/worldgen/structure/vanilla_all_structures`（结构标签）
`common:tags/worldgen/structure/vanilla_all_structures`（结构标签）

`common:tags/block/vanilla_all_blocks`（方块标签）
· 在 24w19a 之前为 `common:tags/blocks/vanilla_all_blocks`。

`common:tags/item/vanilla_all_items`（物品标签）
· 在 24w19a 之前为 `common:tags/items/vanilla_all_items`。

`common:tags/entity_type/vanilla_all_entity_types`（实体类型标签）
· 在 24w19a 之前为 `common:tags/entity_types/vanilla_all_entity_types`。

`common:tags/enchantment/vanilla_all_enchantments`（魔咒标签）
· 加入于 1.20.5-pre1。

`common:tags/potion/vanilla_all_potions`（药水效果标签）
· 加入于 26.1-snapshot-1。

## **静态版本**

每个 Minecraft 版本在数据包里都有自己的版本文件夹。例如：

```
Minecraft Datapack Common Library (Basic Tags Static Edition).zip
├───23w18a
├───1_20-pre1
├───1_20-pre2
...
├───26_3-snapshot-4
│                         
└───pack.mcmeta
```

由于叠加目录的命名规则，Minecraft 版本中所有的 `.` 都转换为 `_`。对应的 pack.mcmeta 如下所示：

```json
{
    "pack": {
        "description": "Minecraft 数据包通用库 1.0.0（基本标签静态版本），支持 23w18~26.3-snapshot-4。",
        "min_format": 15,
        "max_format": 111,
        "supported_fortmats": [15, 111]
    },
    "overlays": {
        "entries": [
            { // 用于 23w18a 的通用库
                "directory": "23w18a",
                "min_format": 15,
                "max_format": 15,
                "formats": [15, 15]
            },
            { // 用于 1.20-pre1 的通用库
                "directory": "1_20-pre1",
                "min_format": 15,
                "max_format": 15,
                "formats": [15, 15]
            },
            { // 用于 1.20-pre2 的通用库
                "directory": "1_20-pre2",
                "min_format": 15,
                "max_format": 15,
                "formats": [15, 15]
            },
            // 省略其它版本
            { // 用于 26.3-snapshot-4 的通用库
                "directory": "26_3-snapshot-4",
                "min_format": [111, 0],
                "max_format": [111, 0],
                "formats": [111, 111]
            }
        ]
    }
}
```

这样子的优点是构建简单，每个版本可独立更新，互不干扰。对于使用相同数据包版本但数据包内容/结构发生改变的 Minecraft 版本来说，对一个版本的修改不会影响到其它版本。缺点是生成的通用库会很大，通用库加载时很多文件都用不上。

标准构建的通用库也是静态版本。

## **动态版本**

多个内容完全相同的版本共用一个文件夹，以优化空间。例如：

```
Minecraft Datapack Common Library (Basic Tags Dynamic Edition).zip
├───23w18a
├───1_20-pre1_to_23w41a
├───23w42a
...
├───26_3-snapshot-3_to_26_3-snapshot-4
│                         
└───pack.mcmeta
```

对应的 pack.mcmeta：

```json
{
    "pack": {
        "description": "Minecraft 数据包通用库 1.0.0（基本标签动态版本），支持 23w18a~26.3-snapshot-4。",
        "min_format": 15,
        "max_format": 111,
        "supported_formats": [15, 111]
    },
    "overlays": {
        "entries": [
            { // 23w18a 的文件和其它版本的文件不完全相同（缺少文件/多出文件/文件内容不一样，下同）
                "directory": "23w18a",
                "min_format": 15,
                "max_format": 15,
                "formats": [15, 15]
            },
            { // 1.20-pre1~23w41a 的文件完全相同，所以共用一个文件夹
                "directory": "1_20-pre1_to_23w41a",
                "min_format": 15,
                "max_format": 20,
                "formats": [15, 20]
            },
            { // 23w42a 和上个版本的文件不完全相同，所以单独一个文件夹
                "directory": "23w42a",
                "min_format": 21,
                "max_format": 21,
                "formats": [21, 21]
            },
            // 省略其它版本
            { // 26.3-snapshot3~26.3-snapshot-4 的文件完全相同，所以共用一个文件夹
                "directory": "26_3-snapshot-3_to_26_3-snapshot-4",
                "min_format": [110, 0],
                "max_format": [111, 0],
                "formats": [110, 111]
            }
        ]
    }
}
```

这样的优点就是体积小，通用性高（一个文件夹可用于多个版本）。缺点就是构建难度高（先生成文件列表并计算每个文件的哈希，再比较优化，耗时更长）、独立性差（修改一个版本会影响到其它版本，特别是相同数据包版本但数据包内容/结构不同的 Minecraft 版本）。

实际上，构建动态版本时，如果将要生成的文件已在标准构建中存在，那么会直接拷贝过去而不会重新生成。这是构建动态版本时的一个优化。

## **注册表**

用于注册各种数据的集合，以便游戏能识别并根据其类型执行对应代码或逻辑。

Minecraft 中广泛使用注册表来注册游戏中几乎所有的元素，并且有些注册表可以通过数据包修改。例如，本文档出现的 block、item、worldgen/structure 等都是可修改的注册表。

参见 [注册表](https://zh.minecraft.wiki/w/注册表)。
